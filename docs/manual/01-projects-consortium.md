# 01. 프로젝트 · 컨소시엄

## 프로젝트 (`/projects`)

### 페이지 구조

| 영역 | 기능 |
|---|---|
| 상단 헤더 | [+ 등록] 버튼 + 상태 필터 + 검색 |
| 목록 | 프로젝트 카드 (이름·상태·기간·고객사·예산) |
| 클릭 | `/projects/:id` 상세 (탭 8개) |

### 신규 등록 흐름

```
1. [+ 등록] 클릭
2. 기본정보 — 이름·고객사(필수)·시작일·종료일
3. 계약 영역 — 계약명·계약금액·단계(proposal/contract/operation/closing)
4. 저장
   → income_contracts 자동 생성 (lifecycle_stage='proposal')
   → 사이드바 "수입" 메뉴에 계약 행 즉시 노출
```

### 상세 페이지 — 탭 8개

| 탭 | 컴포넌트 | 역할 |
|---|---|---|
| 개요 | `OverviewTab` + `ProgramOverviewCard` + 단계별 진행 + 재무요약 + 빠른 액션 |
| 프로그램 | `ProjectProgramsTab` | 묶인 프로그램 목록 + [+ 신규] |
| 태스크 | `TasksTab` | AI 자동 생성 + 그룹별 (사전·현장·사후) |
| 인력 | `MembersTab` | profiles + staff_pool + PM 자동 합류 |
| 재무 | `OverviewTab` 재무요약 + `GrantLedgerTab` |
| 보조금 | `GrantLedgerTab` | 보조금 사업 전용 (있는 경우만) |
| 문서 | `docs/*` | 출석 import·결과물·최종보고 AI 버튼 |
| 포털 | `PortalTab` | 고객 문서 포털 발급 |

### 자주 하는 일

| 상황 | 어디서 |
|---|---|
| PM 변경 | 상세 → 개요 → PM 인라인 편집 |
| 계약 단계 변경 | 상세 → 수입 → 계약 행 클릭 → drawer |
| 강사료 미정산 확인 | 상세 → 재무요약 → "지출 (외주·강사료)" |
| 결과보고서 작성 | 상세 → 문서 → [최종보고 AI 생성] |

### 라이프사이클 5탭 (수입 페이지 기준)

```
🔵 자동      = auto_created=true, 사용자 입력 전
🟡 제안      = proposal
🟣 계약      = contract
🟢 진행      = operation
🟤 종료      = closing
```

⚠️ **알아둘 점** — 프로젝트 만들면 계약은 자동 생성되지만 단계는 `proposal` 시작. 박경수님이 [계약 단계 변경] 으로 진행.

## 컨소시엄 (`/consortium`)

### 페이지 구조

| 영역 | 기능 |
|---|---|
| 상단 | [+ 등록] + 검색 |
| 목록 | 컨소시엄 카드 (이름·의뢰기관·기간·총예산) |
| 클릭 | `/consortium/:id` 상세 (탭 8개) |

### 7+1 탭 구조 — STEP-CON

| 탭 | 컴포넌트 | 역할 |
|---|---|---|
| 개요 | `ConOverviewTab` | 의뢰기관 카드 + 참여기관 지분율 + 예산 집행 |
| 참여사 | `ConMembersTab` | 참여사 등록·수정 + 지분율 합계 경고 |
| 프로그램 | `ConProgramsTab` | 묶인 프로그램 목록 |
| 프로그램 배정 | `ConProgramAssignTab` | 참여사 ↔ 프로그램 다대다 배정 |
| 태스크 | `ConTasksTab` | tasks.consortium_id 필터 + 기관별 share_pct |
| 인력 | `ConStaffTab` | consortium_staff (강사·TA·코디네이터·멘토) |
| 재무 | `ConFinanceTab` | 기관별 수입·지출·잔액 |
| 외부공유 | `ConLinksTab` | 토큰 기반 4종 외부 링크 |
| 포털 | `ConPortalTab` | consortium_portal_permissions (PARTNER) |

### 의뢰기관 vs 운영사 vs 참여사 (STEP-CONSORTIUM-REDESIGN A안)

```
┌──────────────────────────────────────────────────┐
│ 의뢰기관 (lead_client_id)                          │
│  └── 발주자. 예) 전라남도관광재단, 교육지원청       │
│      검수·감수 역할.                              │
└──────────────────────────────────────────────────┘
            ↓ 발주
┌──────────────────────────────────────────────────┐
│ 운영사 (consortium_members 중 role='총괄')         │
│  └── 사업 총괄. 예) 밸런스닷                       │
└──────────────────────────────────────────────────┘
            ↓ 분배
┌──────────────────────────────────────────────────┐
│ 참여사 1·2·3... (role='참여')                     │
│  └── 분담 기관. 지분율로 예산 자동 배분              │
└──────────────────────────────────────────────────┘
```

⚠️ **밸런스닷(자사) 위치 자유화 (2026-05-27)** — 운영사로 들어가도 되고 참여사 중 하나로 들어가도 됨. `clients.is_own_company=true` 행만 있으면 됨.

### 자주 하는 일

| 상황 | 어디서 |
|---|---|
| 참여사 등록 | 상세 → 참여사 → [+ 등록] → 모달 |
| 지분율 합계 100% 확인 | 참여사 탭 상단 합계 영역 |
| 참여사 ↔ 프로그램 배정 | 프로그램 배정 탭 |
| 컨소시엄 포털 발급 | 포털 탭 → 권한 부여 (관리/입력/읽기) |

## 고객사 (`/clients`) — 4종 한글 분류 (2026-05-28)

| 분류 | 의미 |
|---|---|
| **주관기관** | 발주처·의뢰기관 |
| **수혜기관** | 학교·지역 등 사업 수혜자 |
| **참여사** | 컨소시엄 참여사 |
| **거래처** | 일반 거래처·외주처 |

- 자사 (`is_own_company=true`) 는 별도 표시. 등록은 1개만.
- 필터 탭 — 전체 / 주관 / 수혜 / 참여사 / 거래처 / 자사

### 자주 하는 일

| 상황 | 어디서 |
|---|---|
| 신규 등록 | [+ 등록] → AI 명함인식 또는 직접 입력 |
| 4종 분류 변경 | 카드 → 우측 슬라이드 → 분류 인라인 편집 |
| 자사 등록 | [+ 등록] → "자사로 등록" 체크 |

⚠️ **자사는 삭제 불가** — 컨소시엄 자사 기준 행이라 안전 차단.
